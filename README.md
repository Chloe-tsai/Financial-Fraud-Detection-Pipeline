# 📊 公司財務異常預警系統  
(Financial Fraud Detection Pipeline)

建立一套自動化資料管線，從公開資料來源取得上市櫃公司財報數據，透過資料建模與異常偵測，提供企業風險預警機制。

---

## 📌 前言

隨著企業規模擴大與交易複雜化，財務異常與舞弊風險逐漸提高。  
傳統人工稽核方式不僅耗時，亦難以及時發現潛在問題。

因此，本專題以「上市櫃公司財務異常預警」為核心，  
結合資料工程與分析方法，建立自動化監控與預警系統，  
以提升財務透明度與風險控管能力。

---

## 📊 分析主題  
### 上市櫃公司財務異常偵測

**分析內容：**

- 📈 營收異常變動  
- 💸 成本與費用異常  
- 📉 利潤率異常波動  
- 📊 財務比率異常（ROE、毛利率）  
- 🔁 連續期間異常趨勢

## 🧩 系統設計說明（What / Why / How）

---

### 2.1 What：你要做什麼？

建立一套**自動化資料管線（Data Pipeline）**，  
從政府開放平台與金融資訊來源抓取上市櫃公司財報數據  

並計算：

- 📊 Beneish M-Score（會計操縱偵測模型）  
- 📉 Altman Z-Score（破產預測模型）  

👉 當指標異常時，自動觸發預警 🚨  

---

### 2.2 Why：為什麼要做？

#### 🔍 動機
財報舞弊（如博達案、康友案）通常有跡可循，  
但資料分散在 PDF / XML，人工比對極耗時。

#### 💡 價值
- 審計：持續監控（Continuous Monitoring）  
- 投資：過濾地雷股  

#### ⚠️ 技術痛點
- 格式不一致  
- 財報可能更正  
- 需跨期計算  
- 數據驗證困難  

👉 因此需要穩健的 ETL 架構

---

### 2.3 How：要怎麼做？
---

### 🔹 1. 資料獲取（Extraction）

**來源：**
- MOPS API / CSV  
- Yahoo Finance  

**技術：**
- Python（Requests / BeautifulSoup）

**Bronze Layer：**
- 儲存原始 JSON / CSV  
- 存入 Data Lake（S3 / GCS）  

---

### 🔹 2. 資料清洗與轉換（Silver Layer）

**工具：**
- Airflow  
- dbt / Pandas  

📌 Schema 統一  
📌 指標計算：

\[
DSRI = \frac{(\text{應收帳款}_{t} / \text{營收}_{t})}{(\text{應收帳款}_{t-1} / \text{營收}_{t-1})}
\]

👉 DSRI 過高 = 可能營收灌水  

📌 資料驗證：
- 資產 = 負債 + 權益  
- 不平衡 → Alert 🚨  

---

### 🔹 3. 資料建模與呈現（Gold Layer）

**資料庫：**
- PostgreSQL / BigQuery  

📌 SCD Type 2  
👉 保留財報歷史版本（舞弊關鍵🔥）

📌 視覺化：
- Grafana / Power BI  

👉 M-Score > -1.78 → 高風險 🔴
